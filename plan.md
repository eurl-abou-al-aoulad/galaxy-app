## 1️⃣ مساحة Supabase وسعة 150 مستخدم

**الباقة المجانية لـ Supabase (اللي خدامين بيها حالياً):**
- 💾 **500 MB قاعدة بيانات** + 1 GB تخزين ملفات
- 👥 **50,000 مستخدم مفعّل شهرياً (MAU)** — أي 150 مستخدم لا شيء عليها
- 🔁 5 GB نقل بيانات/شهر
- ⚡ 500K استدعاء Edge Functions/شهر
- ⚠️ المشروع يدخل في وضع pause إذا ما تستعملوش 7 أيام (يرجع تلقائياً عند أول طلب)

**حساب واقعي لـ 150 مستخدم/زبون:**
| البند | استهلاك تقديري |
|---|---|
| كل زبون: 500 فاتورة + 200 منتج + 100 زبون = ~2 MB | × 150 = **~300 MB** ✅ |
| Edge Functions (تفعيل + تحقق + sync) | ~50K/شهر ✅ |
| نقل بيانات | ~2 GB/شهر ✅ |

**الخلاصة:** الباقة المجانية كافية لـ 150–200 زبون. عند التجاوز → باقة Pro بـ **$25/شهر** (8 GB قاعدة + 100 GB نقل + 100K MAU).

> 💡 نقدر نضيف لك صفحة في **لوحة المالك** تعرض الاستهلاك الحالي (عدد الصفوف في cloud_sync_items + حجم تقريبي) باش تراقب الوضع لحظة بلحظة.

## 2️⃣ تطبيق الهاتف (Android — Google Play بـ$25)

**الوضع الحالي:** عندك مسبقاً البنية التحتية:
- ✅ `galaxy_mobile_sessions` table موجودة في Supabase
- ✅ `MobileQRPanel.tsx` + `mobile-gateway` edge function
- ✅ صفحة `/mobile` و `/m` للويب موبايل
- ✅ نظام token + license_code + control_code جاهز

**ما يلزم بناؤه:**

### أ) تطبيق Android أصلي — استعمال **Capacitor** (يلف مشروع React الموجود)
```
المخرجات: Galaxy.apk + Galaxy.aab (للنشر على Play Store)
المسار: capacitor-mobile/
```
- يستفيد من 100% من كود `/mobile` الموجود
- يضيف القدرات الأصلية: **الكاميرا** (لمسح QR والباركود) + **الإشعارات** + **التخزين المحلي**
- يربط بـ Supabase نفسها بدون أي تغيير في الـ backend

### ب) ربط QR Code (موجود نظرياً، نكمل التنفيذ)
1. **في برنامج الديسكتوب**: زر "ربط هاتف جديد" يولد QR يحتوي على `{token, license_code, supabase_url}`
2. **في تطبيق الهاتف**: يفتح الكاميرا → يقرأ QR → يحفظ الـ token → يصبح مرتبط بنفس الترخيص
3. **مزامنة فورية**: الهاتف يقرأ من نفس `cloud_sync_items` المرتبطة بـ`license_code`

### ج) خصائص تطبيق الهاتف
- 📦 **مخزون**: عرض + بحث + تعديل كميات
- 💰 **ديون**: قائمة الديون + تسجيل دفعة + إشعار تذكير
- 👥 **زبائن**: قائمة + تفاصيل + سجل المعاملات
- 📄 **فواتير**: قائمة + تفاصيل + إنشاء فاتورة جديدة (sync مع الديسكتوب)
- 📷 **مسح باركود**: عبر كاميرا الهاتف باستعمال `@capacitor-community/barcode-scanner`
- 🔔 **إشعارات**: تنبيه عند بيع جديد / دين قارب أجله

### د) تحضير النشر على Google Play
- إعداد `android/app/build.gradle` (versionCode, signing)
- توليد keystore للتوقيع
- إعداد ملف `play-listing/` (وصف، أيقونات، screenshots)
- دليل خطوة بخطوة لرفع `.aab` على Play Console

## 3️⃣ توليد QR لكل زبون عند التفعيل

**الفكرة:** بعد تفعيل البرنامج بكود `GLXY-XXXX` → البرنامج يولد تلقائياً QR ربط الهاتف يخص هذا الترخيص.

**التنفيذ:**
- إضافة زر "📱 ربط هاتف" في صفحة الإعدادات
- يستدعي `mobile-gateway` لإنشاء session → يرجع token
- عرض QR (باستعمال `qrcode` package) يحتوي:
  ```json
  {
    "v": 1,
    "url": "https://konrwuvpltinzjtjtzsy.supabase.co",
    "license": "GLXY-XXXX",
    "token": "abc123...",
    "shop": "اسم المحل"
  }
  ```
- صلاحية الـ session: 30 يوم (متجدد تلقائياً عند الاستعمال)
- إمكانية ربط عدة هواتف (حسب `mobile_max_devices` في جدول licenses)

## 4️⃣ 🚨 المشكل الخطير: حفظ البيانات بعد انتهاء الترخيص

**الوضع الحالي (بعد فحص الكود):**

✅ **خبر زين:** البيانات **لا تُحذف** عند انتهاء الترخيص! الكود الموجود في `markRevoked()` و `markExpired()`:
- يمسح فقط **توكن الترخيص** (`clearLicense()` يحذف `__galaxy_lic_v1` من localStorage)
- يضع `activated=0` في جدول التفعيل
- **لا يلمس** أي جدول من: invoices, products, customers, debts, expenses, suppliers...

⚠️ **لكن المشكل الحقيقي:** المستخدم لا يقدر **يصل** للبيانات لأن `LicenseGuard` يحجب كامل الواجهة بشاشة حمراء.

### الحل المقترح: وضع "للقراءة فقط" (Read-Only Mode) عند انتهاء الترخيص

بدل الشاشة الحمراء الكاملة، نضمن:

1. **🔒 ضمانة قانونية صريحة في الكود**: تعليق + اختبار آلي يمنع أي PR مستقبلي من حذف البيانات عند الإلغاء
2. **📊 السماح بالاطلاع على البيانات**: المستخدم يقدر يفتح الفواتير القديمة + الديون + الزبائن **للقراءة فقط**
3. **🚫 منع العمليات الجديدة**: لا فاتورة جديدة، لا تعديل مخزون، لا حذف
4. **💾 زر تصدير شامل**: يصدر كل البيانات في ملف `.json` و `.xlsx` يقدر المستخدم يحفظهم محلياً
5. **☁️ ضمانة سحابية**: البيانات في `cloud_sync_items` على Supabase تبقى **30 يوم على الأقل** بعد انتهاء الترخيص (نضيف cleanup job يحذف فقط بعد هذه المدة)
6. **🔓 رسالة واضحة**: "اشتراكك انتهى. بياناتك محفوظة بأمان. جدد التفعيل للاستمرار في العمل العادي." + زر واتساب للدعم

### تفاصيل تقنية للحماية الدائمة

```
src/lib/data-safety.ts (جديد):
  - DATA_NEVER_DELETE_TABLES = ['invoices','products','customers',...]
  - assertNoDestructiveOperationOnLicenseChange()
  - تعليق ضخم في الأعلى يحذر أي مطور مستقبلي
```

```
LicenseGuard.tsx (تعديل):
  - بدل الحجب الكامل → عرض banner أحمر علوي
  - تمرير prop `readOnly={true}` للسياق
  - تعطيل أزرار "حفظ/جديد/حذف" في كل النماذج
```

```
AppContext.tsx (تعديل):
  - إضافة isReadOnly: boolean في الـ context
  - كل دوال الكتابة تتحقق منها قبل التنفيذ
```
