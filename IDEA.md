# 💡 الفكرة — Galaxy Smart Shop Management

## 🎯 المشكلة التي يحلّها البرنامج

أصحاب المحلات الصغيرة والمتوسطة (محلات الموبايلات، الإلكترونيات، الإكسسوارات،
ورش الصيانة، البقالات...) يحتاجون نظام إدارة:

- ✅ **بسيط** يستخدمه أي شخص دون تدريب.
- ✅ **يعمل بالعربية** كلغة أصلية (RTL، خطوط، تواريخ هجرية...).
- ✅ **يعمل أوفلاين** لأن الإنترنت قد ينقطع في أي لحظة.
- ✅ **رخيص الاشتراك** مقابل البرامج المعقدة.
- ✅ **محمي ضد القرصنة** حتى لا يُسرق ويُوزَّع مجاناً.

البرامج الموجودة إما:
- معقّدة جداً (SAP, Odoo) ولا يفهمها صاحب المحل.
- تعمل على الإنترنت فقط فتنقطع خدمة المحل.
- مفتوحة وتُسرَق بسهولة بعد البيع.

## 🌟 الحل — Galaxy

نظام **هجين (Hybrid)** يجمع:

### 1. واجهة عربية فاخرة
- تصميم Galaxy بنمط نجوم متحركة وأيقونات نيون.
- RTL كامل، خطوط عربية واضحة.
- وضع داكن افتراضي مريح للعينين أثناء العمل الطويل.

### 2. وحدات شاملة
- 🧾 الفواتير (إصدار، تعليق، استرجاع، طباعة حرارية).
- 📦 المخزون (باركود، تنبيهات نفاد، تتبع ربح).
- 👥 العملاء والموردين (تاريخ كامل لكل عميل).
- 💰 الديون (متابعة المتأخرات).
- 💸 المصروفات اليومية.
- 🔧 خدمات الصيانة (طلبات إصلاح بحالات).
- ✅ المهام اليومية.
- 📊 لوحة معلومات بإحصائيات لحظية.

### 3. تشغيل أوفلاين بقاعدة بيانات محلية
- استخدام **IndexedDB (Dexie)** لتخزين كل البيانات على الجهاز.
- التزامن مع السحابة عند توفر الإنترنت (اختياري).
- لا حاجة لاتصال دائم بالخادم.

### 4. نظام تفعيل ذكي ومحمي 🔐 (الفكرة الأصلية)
**هنا تكمن العبقرية:**

```
┌─────────────────────────────────────────────────────────┐
│  المالك يولّد كوداً في لوحته السرية (/owner)              │
│                       ↓                                  │
│  يبيعه للعميل بسعر متفق عليه                              │
│                       ↓                                  │
│  العميل يدخله في شاشة التفعيل                              │
│                       ↓                                  │
│  الخادم يربط الكود ببصمة الجهاز (Device Fingerprint)      │
│                       ↓                                  │
│  يُحفظ ترخيص محلي مشفّر بـ HMAC-SHA256                     │
│                       ↓                                  │
│  البرنامج يعمل أوفلاين حتى تاريخ الانتهاء                  │
└─────────────────────────────────────────────────────────┘
```

**النتيجة:**
- لا يستطيع العميل مشاركة الكود مع أحد (مرتبط بجهازه).
- المالك يتحكم: تمديد، إلغاء، نقل لجهاز آخر.
- البرنامج يعمل أوفلاين بعد التفعيل (ميزة حقيقية للعميل).

### 5. حماية متعددة الطبقات ضد القرصنة 🛡️
**فكرة أصلية أخرى:**

| الطبقة | كيف تعمل |
|--------|----------|
| منع F12 و Ctrl+Shift+I | اعتراض الكيبورد على مستوى Window. |
| كشف فتح DevTools | مراقبة `outerHeight - innerHeight` + توقيت `debugger;`. |
| قفل أحمر فوري | عند الكشف، شاشة كاملة تطلب كود المطوّر. |
| Source Maps محذوفة | مستحيل قراءة الكود الأصلي بعد البناء. |
| Mangling قوي | كل المتغيرات تتحول لأحرف عشوائية. |
| Console معطّل | في الإنتاج لا يطبع شيئاً. |
| String Obfuscation | الثوابت الحساسة مشفّرة Base64/XOR. |
| الأسرار الحقيقية | في Edge Functions على الخادم — لا تصل للمتصفح. |

## 🧠 الفكرة محمية فكرياً

كل العناصر أعلاه — وخاصة:
1. الجمع بين **التفعيل بكود مرتبط بالجهاز** + **التشغيل الأوفلاين** + **التحقق الدوري**.
2. منظومة **الحماية متعددة الطبقات** ضد فتح أدوات المطوّر.
3. **لوحة المالك السرية** المعزولة عن المستخدم العادي.
